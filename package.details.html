resolve(link_resource_type, resource_id, array=None)[source]¶

Resolve a link to a CDA resource.

Provided an array argument, attempt to retrieve the resource from the mapped_items section of that array (containing both included and regular resources), in case the resource cannot be found in the array (or if no array was provided) - attempt to fetch the resource from the API by issuing a network request.

Parameters:
  • link_resource_type – (str) Resource type as str.
  • resource_id – (str) Remote ID of the linked resource.
  • array – (Array) Optional array resource.
Returns:

Resource subclass, None if it cannot be retrieved.

class contentful.cda.fields.FieldOwner[source]¶

Bases: type

Metaclass for the Entry class.

For any subclass (i.e. representing a custom Entry class), ensure the extending class has a valid __content_type__ attribute specified, otherwise raise an exception. In addition, iterate through all of the class attributes, identify any Field-typed attributes and keep those in a dict under the class’s __entry_fields__ attribute.

class contentful.cda.resources.Entry(sys=None)[source]¶

Bases: contentful.cda.resources.Resource

CDA resource of type Entry.

Attributes:

  • fields (dict): Entry fields.

It is possible to define custom Entry models using the following syntax:

class Cat(Entry):
    __content_type__ = 'cat'

    name = Field(Text)
    color = Field(Text)
    lives = Field(Number)
    likes = Field(List)
    birthday = Field(Date)
    best_friend = Field(Link, field_id='bestFriend')

Note

field_id is set explicitly for the best_friend field (which otherwise would be inferred from the field’s, as in the other fields).

Entry constructor.

Parameters:sys – dict containing the resource’s remote system attributes.
Returns:Entry instance.
